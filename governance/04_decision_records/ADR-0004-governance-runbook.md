# ADR-0004: Governance Runbook

Status: Approved
Date: 2026-05-24
Operator: Jeremy

## Decision

AXIOM adopts `governance/01_live_spine/AXIOM_Governance_Runbook.md` as the active procedure for executing terminal-native governance.

## Reason

The streamlined governance model needed an explicit process layer to preserve the useful behavior of the prior chat-era governance cycle while keeping the active file set compact. The runbook defines intake, panel review, advisory council use, decision records, active binding updates, verification, and closeout reporting.

## Impact

Future governance work should use the runbook as the procedural reference. The runbook does not make legacy chat-era files active authority; it defines how to pull useful legacy content forward through approved live-spine updates.

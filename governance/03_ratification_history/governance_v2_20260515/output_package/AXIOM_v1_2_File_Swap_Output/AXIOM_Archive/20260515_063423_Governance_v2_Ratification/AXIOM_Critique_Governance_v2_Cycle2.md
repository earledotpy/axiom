New Attack Surface (Non‑Blocking)

Sanitization Pipeline Failure Handling

§12.6 defines the pipeline requirement but does not define the failure mode: if the local model fails to sanitize a document, the operator could supply the unsanitized original or stall. The rule needs a default‑deny posture (quarantine, log, and require explicit panel motion to proceed). I file SD‑V2‑Cycle2‑001 for implementation‑stage closure; it does not block ratification because the pipeline gate itself exists and the non‑authoritative‑status rule prevents binding‑level reliance.

48‑Hour Window Timer Ambiguity

The §7.9 timer starts from operator‑routed request but this is not stated; an operator could delay routing to compress the response window. In practice, the operator is the physical abstraction layer and not adversarial, so the risk is low. Not a correction item.

Architectural Trigger Spurious Invocation

A party could attempt to characterise a non‑architectural change as architectural to force full advisory review, slowing the cycle. The Evaluator and Critic can check this via the missed‑trigger objection path; the Evaluator retains authority to rule that a proposal is not architectural and therefore §7.8 does not apply (the “may not rule these triggers not engaged” applies only to proposals that are architectural). This is a healthy check. No correction needed.
